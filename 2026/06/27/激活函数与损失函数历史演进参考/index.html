激活函数

发表于2026-06-27|更新于2026-06-27|技术文档
|浏览量:

激活函数与损失函数历史演进参考

配合《PyTorch 深度学习演进学习计划》使用。
本文档完整梳理历史上出现的主要激活函数和损失函数，按时间排列，分析每一个函数的优缺点、为何被提出、为何被取代。

第一部分：激活函数

激活函数的演进史，本质上是一部 “解决梯度消失” 与 “解决神经元死亡” 的技术史。每一次新函数的提出都针对前一代函数的特定缺陷。

1. Step / Heaviside 阶跃函数（1943）

  • 提出者：McCulloch & Pitts (1943) / Rosenblatt 感知机 (1958)
  • 形式：f(x) = 1 if x ≥ 0 else 0
优点 缺点
模拟生物神经元的”全有或全无”放电 不可微——无法用梯度下降训练
简单直观，可实现逻辑运算 输出只有 0/1，无法表达连续概率
感知机收敛定理的基础 仅能解决线性可分问题（XOR 不可解 → 第一次 AI 寒冬）

为什么被取代：需要可微的激活函数来支持反向传播训练多层网络。

2. Sigmoid / Logistic 函数（1986）

  • 提出者：Rumelhart, Hinton & Williams — 反向传播论文 (Nature, 1986)
  • 形式：σ(x) = 1 / (1 + e⁻ˣ)，输出范围 (0, 1)
优点 缺点
✅ 可微且光滑——反向传播成为可能 ❌ 梯度消失——导数最大值 0.25，饱和区趋近 0。深层网络中逐层相乘，浅层梯度 ≈ 0
✅ 输出可解释为概率（适合二分类输出层） ❌ 非零中心——输出恒正，导致梯度单向锯齿更新，收敛慢
✅ 有界输出，防止激活值爆炸 ❌ 计算开销大（指数运算）

历史地位：Sigmoid 是反向传播革命的关键推动者，但梯度消失问题限制了网络深度（超过 3-5 层就难以训练）。

3. Tanh / 双曲正切函数（1989）

  • 提出者：LeCun 等 — Backpropagation Applied to Handwritten Zip Code Recognition (1989)
  • 形式：tanh(x) = (eˣ - e⁻ˣ) / (eˣ + e⁻ˣ)，输出范围 (-1, 1)
优点 缺点
✅ 零中心输出——解决了 Sigmoid 的偏移问题，加速收敛 ❌ 梯度消失仍然存在——饱和区导数仍趋近 0
✅ 导数比 Sigmoid 更陡峭，梯度幅度更大 ❌ 计算开销仍然较大（指数运算）

与 Sigmoid 的关系：tanh(x) = 2σ(2x) - 1，本质是缩放平移的 Sigmoid。在浅层网络中优于 Sigmoid，但梯度消失仍是硬伤。

4. ReLU（整流线性单元）—— 革命性突破（2011）

  • 提出者：Glorot, Bordes & Bengio — Deep Sparse Rectifier Neural Networks (AISTATS 2011)
  • 形式：ReLU(x) = max(0, x)
优点 缺点
✅ 基本消除梯度消失——正半轴导数恒为 1，梯度畅通无阻 ❌ 死亡 ReLU 问题——负半轴导数为 0。一旦神经元落入负区，永不再更新（研究表 明可达 40% 的神经元死亡）
✅ 稀疏激活——负值置零，分离有用特征 ❌ 非零中心输出（同 Sigmoid 的偏移问题）
✅ 计算极快——仅需比较运算，无指数计算

为什么是革命性的：ReLU 简单而有效地解决了”梯度消失”这一困扰学界 20 年的核心问题，使得训练数十层深度网络成为可能，直接推动了 2012 年 AlexNet 的成功。

关键时间线：梯度消失问题最早由 Sepp Hochreiter 在 1991 年的毕业论文中正式识别，该发现直接促成了 LSTM (1997) 的发明。20 年后，ReLU 给出了一个简单到令人吃惊的答案：让正半轴梯度恒为 1。

5. Leaky ReLU（带泄漏的 ReLU，2013）

  • 提出者：Maas, Hannun & Ng — ICML 2013
  • 形式：LReLU(x) = max(αx, x)，α 通常 = 0.01
优点 缺点
✅ 解决死亡 ReLU——负半轴引入小斜率（0.01），神经元不会永久死亡 ❌ α 是固定超参数，需手动调优，未必适合所有任务
✅ 保留 ReLU 正半轴导数为 1 的优点 ❌ 性能提升有限——原论文增益不显著，导致长期未取代 ReLU

为什么没取代 ReLU：手动设定的固定负斜率带来的增益不稳定，且 ReLU 的稀疏性在某些任务上反而是有益的正则化。

6. PReLU（参数化 ReLU，2015）

  • 提出者：He, Zhang, Ren & Sun — Delving Deep into Rectifiers (ICCV 2015)
  • 形式：PReLU(x) = max(αᵢx, x)，αᵢ 是 可学习参数
优点 缺点
✅ 自适应学习负斜率——α 与网络权重一同优化，无需人工设定 ❌ 增加了少量过拟合风险
✅ 几乎零额外计算成本 ❌ 负值处理仍是一条固定斜率直线，缺乏更精细机制

里程碑：使用 PReLU 的 ResNet 在 ImageNet 2012 上达到 4.94% top-5 错误率，首次超越人类水平 (5.1%)。

7. ELU（指数线性单元，2015）

  • 提出者：Clevert, Unterthiner & Hochreiter — ICLR 2016
  • 形式：ELU(x) = x if x > 0 else α(eˣ - 1)（α 通常=1）
优点 缺点
✅ 负半轴软饱和 → 激活均值趋近于零，加速收敛 ❌ 计算开销大——包含指数运算，慢于 ReLU
✅ 抗噪声更强——软饱和对噪声输入的鲁棒性优于 Leaky ReLU 的线性负值 ❌ 负半轴梯度仍趋近 0，极端情况下可能退化
✅ 比 ReLU/LReLU 更快收敛、更高准确率（特别是 5 层以上网络）

衍生：SELU (2017) 通过缩放 ELU 的参数使网络自动维持零均值和单位方差，可替代 Batch Normalization。

8. Swish / SiLU —— 自动搜索的产物（2017）

  • 提出者：Ramachandran, Zoph & Le (Google Brain) — 强化学习自动搜索发现
  • 形式：Swish(x) = x · σ(x) = x / (1 + e⁻ˣ)
优点 缺点
✅ 自门控机制——输入 × Sigmoid(输入)，输出可正可负 ❌ 计算成本高于 ReLU——需要计算 Sigmoid
✅ 光滑且处处可微——无限可微，优化曲面更友好 ❌ 缺乏理论解释（由搜索发现，非理论推导）
✅ 非单调性——负值区域有一个”下凹再上升”的形态（x ≈ -1.28 处最小值），比 ReLU 硬零阈值更具表达能力
✅ 在 ImageNet 和 WMT 上一致优于 ReLU

本质洞察：Swish 的出现标志着激活函数设计从”手工设计”进入”自动搜索”时代。

9. GELU（高斯误差线性单元，2016）—— Transformer 的基石

  • 提出者：Hendrycks & Gimpel (2016)
  • 形式：GELU(x) = x · Φ(x)，其中 Φ(x) 是标准正态分布的 CDF
优点 缺点
✅ 随机正则化解释——按输入大小加权，大的输入趋近 1，小的趋近 0，本质是 Dropout 与 ReLU 的整合 ❌ 计算成本较高（需计算高斯 CDF 或其近似）
✅ 光滑且处处可微 ❌ 负半轴梯度也很小
✅ 在 CV、NLP、语音上全面超越 ReLU 和 ELU

为什么至关重要：GELU 被 BERT (2018)、GPT 系列、ViT、CLIP 等几乎所有现代 Transformer 架构采用，是 2018 年至今 NLP 领域的事实标准激活函数。你的学习计划从第 5 章开始处处会见到它。

10. Mish —— 自正则化非单调函数（2019）

  • 提出者：Diganta Misra (2019)
  • 形式：Mish(x) = x · tanh(ln(1 + eˣ))
优点 缺点
✅ 自正则化——导数在负半轴先上升再缓慢下降，起到隐式正则化作用 ❌ 计算成本最高——同时包含 Tanh 和 Softplus
✅ 无限光滑 ❌ 尚未被大规模工业部署广泛采用
✅ 在多个基准上超越 Swish 和 ReLU

激活函数演进总结

当前格局（截至 2026）：

  • GELU — Transformer 架构的绝对主流选择（BERT、GPT、ViT）
  • ReLU — 卷积网络中最广泛使用的激活函数
  • Swish/SiLU — 在特定任务（如 EfficientNet、RL）中展现出替代潜力
  • Sigmoid/Tanh — 仅用于输出层或特定门控机制（LSTM 的门）

第二部分：损失函数

损失函数的演进史，本质上是 “如何定义好与差” 的不断深化——从简单的误分类计数，到概率建模、鲁棒回归、度量学习、对比学习，再到针对特定任务定制。

1. Zero-One Loss（0-1 损失，感知机时代）

  • 形式：L(y, ŷ) = 𝟙[y ≠ ŷ]
  • 对应模型：Rosenblatt 感知机 (1958)
优点 缺点
直观——就是准确率 ❌ 非凸、不可导——无法用梯度下降优化
可解释性强 ❌ 不提供任何梯度信号——正确与错误之间没有平滑过渡

为什么被取代：0-1 损失是 NP-hard 的组合优化问题。统计学习理论催生了”替代损失”（surrogate loss）的概念——使用凸的、连续可微的损失函数作为 0-1 损失的上界代理。

2. Mean Squared Error / MSE（最小二乘，1805/1986）

  • 起源：Legendre (1805) / Gauss (1809) 的最小二乘法
  • 深度学习中：Rumelhart 等 1986 年反向传播论文中的默认损失
  • 形式：L(y, ŷ) = (y - ŷ)²
优点 缺点
凸函数，处处可微 ❌ 对异常值极其敏感——平方放大
高斯噪声下是 MLE 最优解 ❌ 与 Sigmoid 结合做分类时梯度消失——导数包含 σ’(z)，饱和区趋近 0
对大误差惩罚力度大 ❌ 假设误差服从高斯分布，不适用于分类概率建模

为什么被 Cross-Entropy 取代做分类：核心原因是 梯度消失。MSE + Sigmoid 的梯度包含 σ’(z)（饱和时 → 0），而 Cross-Entropy 的梯度 = (p - y)，误差越大梯度越大，实现了”自调节学习率”。

3. Mean Absolute Error / MAE（L1 损失，1757）

  • 起源：Boscovich (1757) / Laplace (1799)
  • 形式：L(y, ŷ) = |y - ŷ|
优点 缺点
✅ 对异常值鲁棒（线性惩罚而非二次） ❌ 零点不可导（需次梯度）
对 Laplace 分布误差是 MLE 最优解 ❌ 收敛速度可能慢于 MSE

4. Huber Loss（1964）

  • 提出者：Peter J. Huber — Robust Estimation of a Location Parameter
  • 形式：小误差时 = MSE（二次），大误差时 = MAE（线性）
优点 缺点
✅ MSE 和 MAE 的最优折衷——小误差时高效，大误差时鲁棒 ❌ 需要手动调节阈值 δ
✅ 处处可导 ❌ 在分类任务中不如 Cross-Entropy 常见

解决的问题：MSE 对异常值过于敏感，MAE 收敛慢且不可导。Huber 通过分段设计取两者之长。

5. Hinge Loss（SVM，1995）

  • 提出者：Cortes & Vapnik — Support-Vector Networks (1995)
  • 形式：L(y, ŷ) = max(0, 1 - y·ŷ)，y ∈ {-1, +1}
优点 缺点
✅ 凸函数，保证全局最优 ❌ 不输出概率（需额外 Platt Scaling）
✅ 最大间隔分类器——泛化能力强 ❌ 对噪声和标签翻转敏感
✅ 支持核技巧处理非线性可分 ❌ 转折点不可导（需次梯度）
✅ 稀疏性——只有支持向量贡献损失

历史地位：1990 年代末到 2000 年代中期是分类任务的事实标准。但 2012 年 AlexNet（使用 Cross-Entropy）后，深度神经网络逐渐取代 SVM + Hinge Loss。

6. Cross-Entropy Loss（交叉熵，1948 / 1990）

  • 起源：Shannon 信息论 (1948)
  • 深度学习引入：Bridle (1990) 推广 Softmax
  • 形式：L = -Σ y_c log(p_c)，p_c = softmax(z_c)
优点 缺点
✅ 完全消除梯度饱和问题——梯度 = p - y，不含 σ’(z) ❌ 对标签噪声敏感——错误标签会导致模型过度拟合
✅ 等价于 MLE 下的多项分布假设，严格的概率解释 ❌ 输出必须是概率分布形式
✅ 模型越有把握越错，惩罚越大（强梯度信号）

为什么取代 MSE 的核心原因：

对比项 MSE + Sigmoid Cross-Entropy + Softmax
梯度形式 σ’(z) · (σ(z) - y) p - y
饱和区梯度 → 0（几乎无信号） 误差越大 → 梯度越大
分类训练速度 极慢 快

这个差异是深度学习从”深不了”到”深得下去”的关键转折之一。

7. Focal Loss（2017）—— 解决类别不平衡

  • 提出者：Lin, Goyal, Girshick, He, Dollár (FAIR) — RetinaNet (ICCV 2017)
  • 形式：FL(p_t) = -(1 - p_t)ᵞ log(p_t)
优点 缺点
✅ 直接解决类别极端不平衡——调制因子降低易分类样本的 loss 贡献 ❌ 引入超参数 γ 需调参
✅ RetinaNet 首次让一阶段检测器超越两阶段检测器 ❌ 主要设计用于密集目标检测
✅ 无需硬负样本挖掘

解决的问题：标准 Cross-Entropy 在密集检测中被大量”易分类负样本”（背景框）的 loss 淹没。Focal Loss 让模型专注于困难的少数样本。

8. Contrastive Loss & Triplet Loss（1993 / 2015）

  • Contrastive Loss：Bromley & LeCun — Siamese 网络 (1993)
  • Triplet Loss：Schroff, Kalenichenko, Philbin — FaceNet (CVPR 2015)
  • 核心思想：学习嵌入空间，同类靠近、异类远离
优点 缺点
✅ 解决开放集识别、人脸验证等分类损失无法处理的任务 ❌ 三元组挖掘极其困难——随机选择无效，全量选择太大
✅ 学到的高质量嵌入可迁移到其他任务 ❌ FaceNet 依赖半难样本挖掘才能稳定训练
✅ 支持 one-shot / few-shot 学习 ❌ 对 batch size 敏感

被 InfoNCE 取代的原因：Triplet Loss 需要复杂的样本挖掘策略（太难的负样 本 → 噪声；太易的负样本 → 无梯度）。InfoNCE 通过 batch 内自然形成负样本对 + softmax 分类形式，无需复杂挖掘。

9. InfoNCE Loss（2018 — 至今最重要的新损失之一）

  • 提出者：van den Oord, Li, Vinyals (DeepMind) — CPC (NeurIPS 2018)
  • 理论基础：Noise-Contrastive Estimation (NCE, 2010)
  • 形式：L = -log[exp(q·k₊/τ) / Σᵢ exp(q·kᵢ/τ)]
优点 缺点
✅ 无需复杂负样本挖掘——batch 内自然形成正负样本对 ❌ 需要大 batch size 或动量编码器来获得足够负样本
✅ 同时利用多个负样本，学习效率高 ❌ 对温度系数 τ 敏感
✅ 有清晰的互信息理论支撑——最小化 InfoNCE = 最大化互信息下界

历史地位：InfoNCE 引 发了对比学习革命。SimCLR (2020)、MoCo (2020) 用它首次在视觉基准上超越有监督预训练。CLIP (2021) 用它实现了强大的零样本跨模态迁移。它 是 2020 年至今自监督学习领域影响力最大的损失函数。

10. KL Divergence（KL 散度，1951 / 2013）

  • 起源：Kullback & Leibler (1951)
  • 深度学习中：Kingma & Welling — VAE (ICLR 2013)
  • 在 VAE 中的作用：Loss = 重构 Loss + D_KL(q(z|x) || p(z))
优点 缺点
✅ 严格的信息论意义 ❌ 不对称性：D_KL(P
✅ 变分推断的数学基础 ❌ 当 Q 在某些区域为零时，KL 发散趋于无穷大
✅ 正则化潜在空间，支持有意义的生成 ❌ VAE 中可能导致后验坍缩

11. CTC Loss（Connectionist Temporal Classification，2006）

  • 提出者：Graves, Fernandez, Gomez, Schmidhuber (ICML 2006)
  • 解决的问题：序列标注中的对齐问题——输入帧比输出标签长得多，且不知道对应关系
优点 缺点
✅ 端到端训练，无需预对齐和音素级标注 ❌ 条件独立性假设——每个时间步的输出条件独立
✅ 天然处理变长输入和变长输出的映射 ❌ 适用于单调对齐，不适用于非单调任务（如翻译）
✅ 推理简单高效

历史地位：CTC 是端到端语音识别的奠基性工作，取代了传统的 HMM-GMM 混合系统。被 DeepSpeech、Whisper 等广泛使用。

损失函数演进总结

核心历史动力：每一次损失函数的演化都是为了解决前一代方法的根本性局限：

从 → 到 解决的问题
0-1 Loss → Hinge/Cross-Entropy 非凸不可导 → 凸可微替代损失
MSE → Cross-Entropy (分类) 梯度饱和 → 梯度与误差成正比
Cross-Entropy → Focal Loss 类别不平衡 → 聚焦困难样本
Cross-Entropy → Triplet/InfoNCE 分类无法学习嵌入 → 度量/对比学习
Triplet → InfoNCE 复杂样本挖掘 → batch 内自然对比

参考文档与你的学习计划对应关系：

  • 第 1-2 章（感知机/MLP）：Step → Sigmoid → Tanh, 0-1 → MSE → Cross-Entropy 的演进
  • 第 3 章（CNN）：ReLU 的重要性在这里显现（深层 CNN 靠 ReLU 才训练得动）
  • 第 5 章（Transformer）：GELU 是 Transformer 的标配激活函数
  • 第 6 章（预训练）：InfoNCE 是 CLIP 的核心损失
  • 第 8 章（多模态）：InfoNCE + Cross-Entropy 的组合
