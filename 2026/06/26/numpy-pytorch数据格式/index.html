bfloat16 和 float32 的指数位都是 8 位 → 两者动态范围完全一样，只是 bfloat16 尾数精度更低（7 vs 23），这是大模型训练选它的核心原因
float8 系列的 e4m3（4指+3尾）和 e5m2（5指+2尾）是互补的：e4m3 精度更高适合权重，e5m2 范围更大适合梯度

数据精度的最低占用是一个字节，是因为内存寻址的最小单位是字节

量化精度（Quantized Precision）

什么是量化？

量化就是将高精度数据（如 float32）映射到低精度整数（如 int8）的过程。核心思想是：用更少的 bit 存，用整数运算法算，尽量保持精度不降太多。

总结

量化精度 = 用低位宽的整数（int8/uint8）配合 scale + zero_point 来近似表示浮点数据，达到 4x 体积压缩 + 2-4x 加速的效果，适用于推理部署场景（移动端、边缘端、生产服务）。PyTorch 的 torch.quantization 模块提供了完整的工具链支持。

动态量化 vs 静态量化：scale / zero_point 的选择

一、动态量化——在线统计

动态量化的 scale 和 zero_point 不是预先算好的，而是在推理时对每一批输入数据实时计算的。

动态量化的完整流程

优缺点

优点 缺点
✅ 无需校准数据，开箱即用 ❌ 每层都要计算 min/max，增加运行时开销
✅ 对输入分布变化鲁棒 ❌ 只有权重被量化（激活值推理时才量化）
✅ 适用于序列模型（LSTM、Transformer） ❌ 加速效果不如静态量化

实际触发时机

动态量化不是每个 token 或每个前向都重新算一次 scale，而是：

  • 对于权重：加载模型时算一次，之后固定
  • 对于激活值：每次推理前（每 batch）算一次

二、静态量化——预校准

静态量化的 scale 和 zero_point 在推理前就确定好了，通过一个”校准（calibration）”步骤。

校准流程

四种常见的校准方法

1. MinMax（默认最简）
  • 直接用 tensor 的全局 min 和 max
  • 实现最简单
  • 问题：对异常值（outlier）非常敏感
  • 如果一个元素是 100，其他都是 0~1，scale 会被拉得很大，导致量化精度损失
2. KL 散度法（熵校准） ⭐ 最常用

这是 TensorRT 和 PyTorch 默认的校准策略（HistogramObserver）：

优势：自动丢弃异常值，保留主要分布的精度。对大模型效果最好。

3. 均方误差法（MSE）
  • 直接最小化量化前后的数值误差
  • 比 KL 散度更直接，但 KL 散度在分类任务上通常表现更好
4. 百分位法（Percentile）
  • 折中方案：去掉明显的异常值，又不像 KL 那样复杂
  • 常用参数：99.9% 或 99.99%

三、两者对比总结

对比项 动态量化 静态量化
scale/zp 计算时机 推理时（每 batch） 校准阶段（推理前）
计算依据 当前 batch 的实际 min/max 校准数据集上统计的分布（KL / MSE / MinMax）
是否需校准数据 ❌ 不需要 ✅ 需要 100~500 个样本
权重 int8（一次性量化） int8（一次性量化）
激活值 float32（推理时才量化） int8（校准时就固定了）
推理加速 1.5~2x 2~4x
精度损失 较小（约 0~1%） 稍大（约 0.5~2%）
适用场景 LSTM / Transformer / 小模型 CNN / 大模型生产部署
PyTorch API quantize_dynamic() prepare() → calibrate → convert()

一句话总结

  • 动态量化：每次来数据现算 min/max → 现场决定 scale/zp → 好处是无需校准，坏处是慢一点点
  • 静态量化：先用一小批数据跑一遍，”观察”每层的激活值分布（KL 散度等算法选最佳阈值）→ 固定 scale/zp → 好处是推理时零额外开销，速度最快
