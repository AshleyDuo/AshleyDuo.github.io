深度学习计划

PyTorch 深度学习演进学习计划

序言

你已经完成了 PyTorch 基础入门，能够在 Mac 上用 MPS 跑起简单的模型。接下来要走的是一条更有意思的路——沿着 AI 架构的历史脉络，亲手还原每一个时代最具代表性的结构。不是泛泛了解，而是用代码实现一遍，跑起来、看到结果、感受它的局限。

每一章结束时，你应该清楚两件事：这个架构比上一个强在哪，以及它自己有什么解决不了的毛病。带着这些”未解决问题”进入下一章，你会自然理解为什么下一个架构长成那样。

第一章 · 感知机 —— 一切的起点

对应年代：1958（Rosenblatt）/ 1969（Minsky & Papert 揭示局限）

你将从零实现一个单层感知机。这不是 PyTorch 练习——手动实现权重更新，不用框架的自动求导。

核心概念

  • 神经元作为线性二分类器
  • 阶跃激活函数
  • 感知机收敛定理
  • 决策边界与线性可分性

要实现的内容

  1. 用纯 Python + NumPy 实现感知机的训练循环（手动计算梯度、手动更新权重）
  2. 在 AND / OR 数据集上验证收敛
  3. 用 PyTorch tensor 重写一遍（感受框架带来的简化）

关键实验

  • XOR 问题：实现感知机，观察它在异或数据集上永远无法收敛。这是历史上导致第一次 AI 寒冬的直接原因。
  • 可视化决策边界的变化过程。

这一章的未尽问题

感知机只能处理线性可分问题。绝大多数现实问题——图像、语言、声音——都不是线性可分的。
怎么办？

第二章 · 多层感知机与反向传播

对应年代：1986（Rumelhart, Hinton, Williams 发表反向传播论文）

加入隐藏层，解决 XOR。手动推导链式法则如何变成反向传播算法。

核心概念

  • 多层网络与隐藏层的表达能力
  • 链式法则与反向传播（手动推导与手动实现）
  • 非线性激活函数（Sigmoid / Tanh）
  • 万能近似定理（Universal Approximation Theorem）

要实现的内容

  1. 手动实现一个 2 层 MLP 的前向和反向传播（纯数学计算，不用 autograd）
  2. 验证它能够解决 XOR 问题
  3. 用 PyTorch nn.Linear + autograd 重写同一网络（感受框架价值）
  4. 在 MNIST 手写数字数据集上训练一个 MLP 分类器

关键实验

  • 移除激活函数，观察多层退化为一层——说明非线性是关键。
  • 改变隐藏层宽度，观察表达能力与过拟合的关系。
  • 梯度消失实验：加深网络层数（4 层、6 层、8 层），用 Sigmoid 激活，观察浅层梯度趋近于零。

这一章的未尽问题

随着网络加深，Sigmoid/Tanh 的梯度消失问题越发严重。更深层的网络无法有效训练。
此外，全连接结构将图像当成平铺向量，完全丢失了空间结构信息。
怎么办？

第三章 · 卷积神经网络 —— 让网络学会”看”

对应年代：1998（LeCun — LeNet-5）/ 2012（AlexNet — ImageNet 转折点）

引入局部连接与权值共享，让网络利用图像的空间结构信息。

核心概念

  • 卷积核与特征提取
  • 权值共享与平移不变性
  • 池化层（下采样）
  • 感受野与层级特征
  • 现代 CNN 的演进脉络

要实现的内容

  1. 手动实现一个二维卷积的前向计算（不借助 PyTorch 的 Conv2d，用 unfold 或 im2col 理解其本质）
  2. 用 PyTorch 从零搭建 LeNet-5 并在 MNIST 上训练
  3. 实现 AlexNet / VGG 的简化版，在 CIFAR-10 上训练并观察性能提升
  4. ResNet：实现残差连接（Skip Connection），在 CIFAR-10 上训练一个较深的 ResNet（ResNet-18/34），验证它能够缓解梯度消失

关键实验

  • 可视化每一层卷积学到的特征图，观察低层（边缘、纹理）到高层（部件、物体）的层级抽象。
  • 对比同深度的 Plain Network 与 ResNet，观察训练 loss 的差异——残差连接如何让更深的网络变得可训练。
  • 不使用 MPS，纯 CPU 训练你的第一个 CNN，感受计算瓶颈。

这一章的未尽问题

CNN 需要固定尺寸输入（全连接层限制），且难以处理序列数据（文本、语音、时间序列）。
卷积核的感受野是局部的，对长距离依赖关系建模困难。
怎么办？

第四章 · 循环神经网络与序列建模

对应年代：1997（Hochreiter & Schmidhuber — LSTM）/ 2014（Cho — GRU）/ 2014（Sutskever — Seq2Seq）

让网络拥有”记忆”，处理变长序列数据。

核心概念

  • 循环连接与隐状态
  • BPTT（Backpropagation Through Time）
  • 长期依赖问题与梯度爆炸/消失
  • LSTM 的门控机制（遗忘门、输入门、输出门、细胞状态）
  • GRU 的简化设计
  • Seq2Seq 架构

要实现的内容

  1. 手动实现一个简单 RNN 的前向传播，理解隐状态的循环更新逻辑
  2. 用 PyTorch 实现 LSTM，在时间序列预测任务（如 sin 波预测）上训练
  3. 名字分类 / 名字生成任务：使用字符级 RNN/LSTM 进行分类或文本生成（PyTorch 官方教程中的经典项目）
  4. 实现 Seq2Seq 模型用于简单机器翻译或加法序列任务

关键实验

  • 对比简单 RNN 与 LSTM 在长序列上的 loss——观察简单 RNN 的梯度消失如何导致早期信息丢失。
  • 调整序列长度（10 vs 50 vs 200），观察 LSTM 的遗忘曲线。

这一章的未尽问题

RNN 及其变体是顺序计算的——无法并行。序列越长，训练越慢。
长距离依赖理论上由 LSTM 解决，但在实践中超过几百步仍然困难。
此外，信息被压缩在一个固定大小的隐状态向量中——这是一个严重的瓶颈。
怎么办？

第五章 · 注意力机制与 Transformer

对应年代：2015（Bahdanau Attention）/ 2017（Vaswani — “Attention Is All You Need”）

抛弃循环结构，用注意力机制直接建模序列中任意两个位置的关系。

核心概念

  • Bahdanau Attention vs Luong Attention
  • 自注意力（Self-Attention）与缩放点积注意力
  • 多头注意力（Multi-Head Attention）
  • 位置编码（Positional Encoding）
  • Transformer 的 Encoder-Decoder 完整结构
  • 并行计算的优势与二次复杂度的代价

要实现的内容

  1. 从零实现缩放点积注意力机制
  2. 实现多头注意力模块
  3. 搭建一个 mini Transformer（1-2 层），在简单的翻译或序列任务上训练
  4. 关键里程碑：用你实现的 Transformer 做语言建模（如 Character-Level Language Model on Shakespeare / WikiText-2），对比它与 LSTM 的训练速度和生成质量

关键实验

  • 可视化注意力权重矩阵——观察模型在生成某个 token 时关注了哪些位置。
  • 对比 Transformer 与 LSTM 的训练速度（Transformer 可并行 → 更快）。
  • 尝试不同的序列长度，观察 Transformer O(n²) 的内存开销与 LSTM O(n) 的对比。

这一章的未尽问题

Transformer 的 n² 复杂度在处理长序列时非常昂贵。
它是一个自监督预训练结构，但如何利用海量无标注数据？如何在下游任务上有效利用？
而且，它目前只是一个 backbone——能表示，但不会”完成任务”。
怎么办？

第六章 · 预训练时代 —— BERT 与 GPT

对应年代：2018（BERT —— Devlin）/ 2018（GPT / GPT-2 —— Radford）/ 2020（GPT-3 —— Brown）

从”为每个任务训练一个模型”转向”预训练 + 微调”。大规模无监督预训练改变了整个领域。

核心概念

  • 自监督学习（Masked Language Model vs 自回归语言模型）
  • BERT 的 Encoder-only 架构与双向上下文
  • GPT 的 Decoder-only 架构与自回归生成
  • 预训练 → 微调范式
  • 迁移学习与零样本/少样本能力

要实现的内容

  1. 实现一个简化版 BERT（约 4-6 层），在 WikiText-2 或 BookCorpus 上做 MLM 预训练
  2. 在 GLUE / MRPC / SST-2 上微调你的 BERT，观察预训练带来的效果
  3. 实现一个简化版 GPT（Decoder-only），在文本生成任务上观察自回归生成
  4. 关键项目：搭建一个 toy GPT，在莎士比亚作品上训练，生成模仿风格的文本

关键实验

  • 消融实验：对比从头训练的模型 vs 预训练后微调的模型在下游任务上的性能差距。
  • 改变预训练数据量（10% / 50% / 100%），观察效果与数据量的关系。
  • 可视化 BERT 中不同层的注意力模式——低层关注句法、高层关注语义。

这一章的未尽问题

这些模型越来越大（BERT-Base 110M，GPT-3 175B）。规模带来能力，但训练成本急剧上升。
预训练 + 微调仍需要每个任务单独微调，GPT-3 展示的 in-context learning 为何发生？更大的模型涌现出什么能力？
模型生成的结果有时看似合理实则错误——幻觉问题。
而且，所有模型都只处理文本。图像、视频、音频怎么办？
怎么办？

第七章 · 生成模型 —— 从 GAN 到扩散模型

对应年代：2014（GAN —— Goodfellow）/ 2020（DDPM —— Ho）/ 2022（Stable Diffusion —— Rombach）

从判别式到生成式。让模型学会创造——生成逼真的图像。

核心概念

  • 生成对抗网络（GAN）：生成器与判别器的对抗训练
  • GAN 的不稳定训练与模式坍塌
  • 变分自编码器（VAE）与 ELBO
  • 扩散模型（DDPM）：前向加噪 → 反向去噪
  • 潜在扩散模型（Latent Diffusion / Stable Diffusion）
  • 分类器无关引导（Classifier-Free Guidance）

要实现的内容

  1. 在 MNIST 上实现一个 GAN（DCGAN），观察对抗训练的动态平衡
  2. 实现一个 VAE，对比 VAE 生成图像的模糊性与 GAN 的锐利性
  3. 实现一个简化版 DDPM（Denoising Diffusion Probabilistic Model），在 MNIST 上训练，可视化去噪过程
  4. 关键项目：用预训练的 Stable Diffusion 模型跑一次 text-to-image 推理（理解它的完整 pipeline：Text Encoder → UNet → VAE Decoder）

关键实验

  • GAN：可视化训练过程中生成器与判别器的 loss 曲线——观察它们如何博弈。
  • GAN：故意让训练不稳定（学习率过高、网络不平衡），观察模式坍塌。
  • DDPM：从一个纯噪声开始，逐 step 可视化去噪过程（理解它为什么不是一步生成）。
  • 对比 GAN 与扩散模型的生成多样性 vs 质量。

这一章的未尽问题

扩散模型生成质量高，但采样速度慢（需要数百步迭代）。
GAN 快但不稳定。这些模型都只能生成图像。文本生成和图像生成还是两个独立的世界。
如何将视觉理解与语言理解统一？视觉信息如何与文本对齐？
怎么办？

第八章 · 多模态 —— 让模型”看懂”世界

对应年代：2021（CLIP —— OpenAI）/ 2023（GPT-4V）/ 2024（LLaVA、Qwen-VL）

将视觉与语言连接起来。CLIP 统一了图像和文本的表征空间，视觉语言模型让 AI 既能看又能说。

核心概念

  • 对比学习与 CLIP（Contrastive Language-Image Pre-training）
  • 图像编码器 + 文本编码器的双塔结构
  • 视觉-语言对齐与跨模态检索
  • LLaVA 架构：视觉编码器 → 投影层 → LLM
  • 多模态思维链与视觉推理

要实现的内容

  1. 理解 CLIP 的训练目标（对比学习 + InfoNCE Loss），用预训练 CLIP 做 zero-shot 图像分类
  2. 用 CLIP 做图文检索（text→image / image→text），观察跨模态对齐效果
  3. 实现一个简化版 LLaVA：用 SigLIP/ViT + 投影层 + 一个小型 LLM（如 Gemma-2B），让模型能够”看图说话”
  4. 将你自己的图片送入模型提问，观察视觉语言模型的理解能力

关键实验

  • CLIP zero-shot：在 CIFAR-100 上不做任何训练，直接用 CLIP 分类，对比它的准确率与监督学习的差距。
  • 用对抗性或模棱两可的图片测试 LLaVA，观察多模态幻觉——模型看到了什么，它说了什么。
  • 分析 CLIP embedding 空间的语义结构（t-SNE 可视化）。

这一章的未尽问题

多模态模型仍然以 Transformer 为核心，视觉编码器 + LLM 拼接的方式并非真正的”原生多模态”。
模型参数量膨胀，推理成本高。幻觉问题在多模态下更加严重——模型会”看见”不存在的东西。
能否既保持性能、又降低推理成本？能否让模型具备真正的推理能力而不是模式匹配？
怎么办？

第九章 · 效率革命 —— 小模型、MoE 与推理范式转变

对应年代：2023~2026（Mixtral / DeepSeek / Phi-3 / Gemma / Qwen2.5 / DeepSeek-R1 / Mamba）

前沿正在发生的事。三条并行的路径同时推进：让模型更小更高效、让专家系统更智能、让推理范式发生根本转变。

三条主线的脉络

路径 A：小模型与数据效率

Phi-3（Microsoft）、Gemma（Google）、Qwen2.5 系列证明了：高质量数据可以弥补参数量的不足。3.8B 参数的模型可以达到 70B 级别模型数年前的水平。这颠覆了”越大越好”的路线。

路径 B：混合专家模型（MoE）

Mixtral 8x7B、DeepSeek-MoE、Qwen2.5-MoE 引入了条件计算——每次推理只激活部分参数。在保持推理效率的同时扩大模型容量。MoE 并非新概念（1991 年提出），但工程上的成熟让它重新成为焦点。

路径 C：推理模型 —— 从模式匹配到”思考”

DeepSeek-R1（2025）和 OpenAI o1 / o3 系列开启了”推理模型”这个新范式。模型在回答之前先进行内部推理（Chain-of-Thought 的显式化），通过强化学习在推理步骤上训练。这不再是简单的下一个 token 预测——模型学会了”停下来想一想”。

路径 D：状态空间模型 —— 超越 Transformer 的尝试

Mamba（2023）用状态空间模型（SSM）替代注意力机制，在长序列任务上实现了 O(n) 复杂度。传统 RNN 的复兴，但通过硬件友好的并行扫描算法解决了训练问题。

要实现的内容（选做，根据兴趣侧重）

  1. 小模型路线：用蒸馏或剪枝技术将一个已训练好的小模型压缩为目标大小的 50%，对比性能损失
  2. MoE 路线：实现一个简化版的 MoE 层（Router + Top-K + 多个 Expert FFN），在 toy 任务上观察负载不均衡问题，并实现辅助 loss 来缓解
  3. 推理模型概念：理解 Chain-of-Thought 的显式化——在 LLM 的生成过程中插入”思考标记”（think tokens），观察模型如何分配推理步骤
  4. 状态空间模型：实现 Mamba 的核心思想——S4 模型的一维状态空间更新，在序列建模任务上对比它与 Transformer 的内存/速度曲线

关键实验

  • MoE：在 toy 路由任务上可视化 Expert 的选择分布——看看 Router 是否学会了有意义的分配。
  • SSM vs Transformer：在长序列（8K tokens）上对比两者的内存占用和训练时间。
  • 推理模型：对比标准 LLM 与加了 CoT 推理步骤的版本在数学推理任务上的准确率差异。

第十章 · 回望与方向

你走过的路

每一代架构都解决上一代的根本局限，同时又引入新的问题。这种”继承—突破—新瓶颈”的循环是深度学习发展的底层节奏。

当你完成后，你可以回答这些问题

  • 为什么 CNN 在 2012 年爆发，但 RNN 始终未能实现类似的飞跃？
  • Transformer 到底解决了 RNN 的什么核心问题？它又带来了什么新问题？
  • 为什么 GAN 没有成为生成图像的主流，反而是扩散模型后来居上？
  • BERT 和 GPT 的架构差异本质是什么？为什么最后 GPT 的路线赢了？
  • 多模态模型的核心挑战是架构问题还是数据问题？
  • 2026 年的”推理模型”范式可能将深度学习带向何方？

持续跟踪前沿的建议

  • Papers with Code（paperswithcode.com）——按任务查看最新 SOTA
  • Hugging Face Daily Papers（huggingface.co/papers）——每天精选 AI 论文
  • Arxiv Sanity / Semantic Scholar —— 论文搜索与推荐
  • The Batch by Andrew Ng——每周一次的深度学习新闻精读
  • GitHub Trending —— 关注 trending repositories 中的 AI 项目
