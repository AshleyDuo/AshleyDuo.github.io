一、博客概况

你的博客是一套基于 Hexo 框架搭建的静态博客，托管在 GitHub Pages 上。

1.1 核心技术栈

项目 内容
博客框架 Hexo 8.1.2
主题 Butterfly 5.5.5（通过 npm 安装）
部署方式 hexo-deployer-git → GitHub Pages
写作语言 Markdown
代码高亮 highlight.js（内置）

1.2 个人配置

配置项 当前值
语言 zh-CN（中文）
时区 Asia/Shanghai

1.3 两个仓库

你的博客涉及 两个 Git 仓库：

  • 源码仓库（即你当前看到的这个目录）：存放 Hexo 源码、文章的 Markdown 文件、主题配置等。相当于”厨房”——所有原材料和工具都在这里。
  • 部署仓库（AshleyDuo/AshleyDuo.github.io）：存放 Hexo 生成的静态 HTML 文件，浏览器实际访问的是这个仓库。相当于”餐厅”——顾客看到的是这里的东西。

每次执行 hexo deploy，Hexo 会自动生成最新页面并推送到部署仓库，不需要手动操作部署仓库。

二、文件目录结构详解

2.1 各文件/目录详细说明

_config.yml — 博客核心配置文件。站点标题、URL、主题选择、部署信息等都写在这里。这是你最需要了解和修改的文件。

package.json — Node.js 项目依赖声明。Hexo 本身、所有插件和主题的版本都在这里。一般不由你手动编辑，而是通过 npm install xxx --save 自动更新。

scaffolds/ — 脚手架模板。当你运行 hexo new post "标题" 时，Hexo 会根据 scaffolds/post.md 生成新文件。你可以修改这些模板，让每篇新文章自带你想要的默认字段。

source/ — 源文件目录。这是你日常写作最常打交道的地方：

  • source/_posts/ 存放所有博客文章
  • 你也可以在 source/ 下创建自定义页面，如 source/about/index.md
  • 建议在 source/images/ 存放图片（目录目前不存在，需要时手动创建）

public/ 和 .deploy_git/ — 自动生成的目录，不需要手动修改。它们已被加入 .gitignore，不会上传到源码仓库。

四、Front-matter 配置

每篇文章顶部的 --- 包裹区域称为 Front-matter，使用 YAML 格式定义文章的元数据。当前 _config.yml 中配置的 URL 格式为 :year/:month/:day/:title/，因此文章 URL 由这些字段决定。

4.1 所有可用字段

字段 必填 说明 示例
title 是 文章标题 title: 我的文章
date 是 发布日期，影响排序 date: 2026-06-25 14:00:00
updated 否 最后修改日期，覆盖文件 mtime updated: 2026-06-26 10:00:00
tags 否 标签，多个用数组 tags: [Hexo, 教程]
categories 否 分类，层级式 categories: [技术, Hexo]
keywords 否 SEO 关键词 keywords: Hexo教程, 博客
description 否 文章摘要/描述 description: 这是一篇入门教程
cover 否 封面图（Butterfly 主题支持） cover: /images/cover.jpg
permalink 否 自定义永久链接 permalink: my-url

4.2 完整示例

4.3 注意事项

  • 日期格式：YYYY-MM-DD HH:mm:ss，中间有空格
  • 分类是层级式的：[技术文档, Hexo] 表示”技术文档 > Hexo”的子分类关系
  • 如果想要多个独立分类：需要用嵌套数组语法，如 categories: [[技术文档], [JavaScript]]
  • 配置文件中有一个拼写错误：_config.yml 第 20-21 行的 trailing_index 和 trailing_html 的值写成了 flase（应为 false）。虽然 Hexo 可能容错，但建议更正。

九、常用命令速查

操作 命令 说明
创建新文章 npx hexo new post "标题" 在 source/_posts/ 生成新文件
创建新页面 npx hexo new page "关于" 生成 source/about/index.md
创建草稿 npx hexo new draft "标题" 在 source/_drafts/ 创建草稿
发布草稿 npx hexo publish "标题" 将草稿移至 source/_posts/
启动本地服务 npx hexo server 访问 http://localhost:4000
指定端口 npx hexo server -p 5000 用 5000 端口
预览草稿 npx hexo server --draft 包含草稿的本地预览
生成静态文件 npx hexo generate 输出到 public/
清理缓存 npx hexo clean 清空缓存和 public/
部署 npx hexo deploy 推送到 GitHub Pages
一键发布 npx hexo clean && npx hexo generate && npx hexo deploy 清理→生成→部署
安装新插件 npm install 插件名 --save 如 npm install hexo-filter-mathjax --save
更新所有依赖 npm update 保持主版本号不变

npx 说明：建议始终使用 npx hexo 前缀，而不是直接 hexo。因为 Hexo 是本地安装的，npx 会自动从 node_modules 中调用它。如果你没有全局安装 hexo-cli，直接 hexo 会提示命令未找到。

十、常见问题

Q1: 修改了内容但浏览器看到的是旧的

运行 npx hexo clean 清除缓存，再 npx hexo generate 重新生成。如果修改了 _config.yml，记得重启 hexo server。浏览器中按 Ctrl+F5（Mac: Cmd+Shift+R）强制刷新。

Q2: hexo deploy 报认证错误

GitHub Token 可能过期了。去 GitHub Token 设置 生成新 Token（勾选 repo 权限），替换 _config.yml 中 repo 链接里的 Token。

Q3: 文章创建了但首页不显示

可能的原因：

  • 文件在 source/_drafts/ 而不是 source/_posts/
  • Front-matter 格式错误（比如冒号用了中文全角 ：）
  • 文件没有 .md 扩展名

Q4: 图片不显示

  • 路径要以 / 开头（如 /images/photo.jpg）
  • 图片必须放在 source/ 目录下，否则不会复制到 public/
  • 确保 source/images/ 目录存在

Q5: hexo server 端口被占用

用 -p 参数指定其他端口：npx hexo server -p 5000

Q6: 如何更新 Hexo 或主题？

注意大版本更新可能有兼容性问题，更新前建议看发布说明。

Q7: 如何从其他电脑上继续写作？

  1. 克隆源码仓库：git clone <仓库地址>
  2. 进入目录，运行 npm install 安装依赖
  3. 确认 _config.yml 中的 Token 仍然有效（不同电脑可能需要重新配置）
  4. 然后就可以写作、预览和部署了
