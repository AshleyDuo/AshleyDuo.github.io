HDLbits 刷题笔记 P1

发表于2026-07-12|更新于2026-07-12|项目笔记

为什么要写这个

笔者之前有多次学习verilog的经历，但因为工作内容不涉及复杂的数字电路设计，因此水平限于入门，最近在一生一芯的学习中，在HDLbits上刷题，看到了很多对笔者来说比较陌生的语法，还有一些常用语法但其实并没有搞清细节。

这篇文章记录了我从 HDLbits 实战中挖出来的、速查表要么没写、要么写了也看不懂为什么有用的语法细节。

1. 操作符 位操作

1.1 位拼接 {}

只用一行 assign 实现 100 位全加器

{c[98:0], cin} 构造了一个 100 位的”进位向量”，低位进位 cin 被塞到第 0 位，高位进位 c[98:0] 依次左移。

没有 generate，没有 for 循环，没有 Full_adder 例化——纯连线和位拼接就把 ripple carry 写完了。

💡 核心思想：{} 不止是拼接信号，更是在构造位宽匹配的中间向量。

1.3 缩减运算符（Reduction operators）

^in 等价于把 in 的每一位全部 XOR 起来。语法上它就是个单目运算符，不是双目。速查表会写”缩减运算符”几个字，但不会告诉你它在实际电路里就是用一串 XOR 树综合出来的，也不会告诉你 ~|in 是检测全 0 最简洁的写法。

1.4 索引部分选择 +: ——动态位宽切片

2. 两种循环：for vs generate for——选错就出错

这是 HDLbits 到 Circuits 阶段最大的分水岭。速查表上两个都有，但什么时候用哪一个才是真问题。

2.1 for 循环——过程性循环，写在 always 里面

这个 for 不是”循环执行 255 次”，它在综合时被完全展开，等价于 255 个加法器排成加法树。always @(*) 里的 for 只是写代码方便，硬件上没有循环。

2.2 generate for——结构性循环，写在 module 级

区别：

特性 always 里的 for generate for
循环变量类型 integer genvar（编译期常量）
展开时机 综合时展开 编译前展开（更早）
能生成什么 块内逻辑 module、assign、always 块
典型场景 popcount、累加、移位 例化 N 个模块、N 个 always
能否嵌套 if-else 可以（if 在循环体内） 可以（用 generate if）

💡 选型指南：

  • 只需要在一个 always 块里重复逻辑 → for in always
  • 需要生成多个独立的 always / module / assign → generate for
  • 需要条件编译（有些位用不同逻辑）→ generate for + generate if

2.3 begin: label_name 不是装饰

没有 begin: adder 会产生编译错误。这个名字作用有二：

  1. 层次化引用：外部可以通过 adder[0].cout、adder[1].sum 引用内部信号
  2. 唯一标识：综合工具用它区分生成的 100 个实例

3. 实战硬核模式

3.1 双边沿触发器（Dual-edge triggered FF）

Verilog 没有 always @(posedge clk or negedge clk) 这种写法——至少对于时钟来说不行（那是异步复位语法）。

怎么办？

工作原理：在 clk 上升沿采一次 d 到 q1，下降沿采一次 d 到 q2，然后用 clk 电平做 mux 选择。clk=1 时输出 q1，clk=0 时输出 q2。这样每个时钟周期输出变化两次，等效双边沿触发。

综合后是 2 个 DFF + 1 个 2-to-1 mux，不是真正的”双边沿 DFF”（那个要定制库才有）。

3.3 有符号数溢出检测

逻辑：两个正数相加得负数，或者两个负数相加得正数 → 溢出。

a[7] b[7] s[7] overflow 含义
0 0 0 0 正+正=正 ✅
0 0 1 1 正+正=负 ❌ 溢出
1 1 1 0 负+负=负 ✅
1 1 0 1 负+负=正 ❌ 溢出
0 1 X 0 异号相加不可能溢出

对比无符号数的进位检测（cout），有符号数的溢出要看你想要哪种保护。

3.4 边界绕回——Verilog 不会帮你自动 wrap

C 语言里 array[-1] 会访问到数组前面一个位置（可能 crash），Verilog 里 in[-1] 直接给你 X。没有自动回绕，没有模运算——你必须显式处理边界。

正确做法：三目运算符做边界判断，或者用 (i - 1 + N) % N（记得先加 N 再取模，因为 Verilog 的 % 对负数结果是负数）。

4. 综合实战中才会摸到的模式

4.1 LFSR(Linear Feedback shift register)——伪随机数生成器

LFSR 只需要几个 XOR 门就能生成周期为 2^N − 1 的伪随机序列。它的关键参数是抽头多项式（taps），决定了最大周期。上面的 32-bit LFSR 使用的是 x^32 + x^22 + x^2 + x^1 + 1 多项式——这是 IEEE 标准里定义的。

对比二进制计数器的全加器进位链，LFSR 是时序逻辑中最省门的结构之一。
